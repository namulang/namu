147  // check whether sub iter has been reached to reversed non-boundary end iter.
148  // e.g.
149  // chain A has {1, 2, 3} elements.
150  // chain B has {4, 5, 6} elements.
151  // when A linked B from the 2nd element, chains will be follow:
152  // {1, 2, 3, 5, 6}
153  // in this case, what if user wants to iterate in reversed?
154  // then elements to be out should be:
155  // {6, 5, 3, 2, 1}
156  // our sub iterate can detect end of the sub container inside of its logic when
157  // you call `next()` in `_iterate()`.
158  // however, in above example, after sub iter reached to `5`, it's not possible for it
159  // to detect the end of iteration of the sub container. because actually there is one
160  // more element to iterate, `4`.
161  // so only who is capable of knowing that reversed iteration reached to the end for
162  // reversed iterator is need to be done by `nchainIteration` class.
163  // and this func is for that.
